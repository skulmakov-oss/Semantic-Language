# E0242: re-export collision

## What it means

Two re-export paths in one module resolve to the same public name.

Typical diagnostic includes wording like:
`re-export collision for 'X'`.

## Example

```exo
Import pub "a.sm"
Import pub "b.sm"
```

If both modules export `Sensor`, the current module collides on the public
re-export name `Sensor`.

## How to fix

1. Rename one symbol via `as`.
2. Switch to selective re-export (`{ Foo }`) instead of full/wildcard.
3. Keep only one source of truth for each exported public name.

## Notes

Re-export ordering is deterministic, but collisions are still hard errors.
