# E0242: re-export collision

## What it means

Two exports in one module resolve to the same public name.

Typical diagnostic includes wording like:
`export name collision for 'X'`.

## Example

```exo
Import pub "a.exo"
Import pub "b.exo"
```

If both modules export `Sensor`, the current module collides on `Sensor`.

## How to fix

1. Rename one symbol via `as`.
2. Switch to selective re-export (`{ Foo }`) instead of full/wildcard.
3. Keep only one source of truth for each exported public name.

## Notes

Export ordering is deterministic, but collisions are still hard errors.
