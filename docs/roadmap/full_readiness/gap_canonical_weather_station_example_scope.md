# GAP: Canonical Weather Station Example and Golden Vector

Status: implementation-scope draft  
Track: Semantic Full Readiness / canonical examples  
Source trigger: successful `weather_station.sm` trial

## Problem

The weather-station program is now a useful non-trivial Semantic example, but it is not yet captured as a canonical repository artifact.

Without a canonical location, README, and golden vector policy, the example remains an external experiment rather than a repeatable readiness proof.

## Goal

Promote the weather-station decision program into a canonical example that demonstrates:

- `quad` classification;
- `f64` decision logic;
- composed status aggregation;
- suite-style self-checking;
- compile to `.smc`;
- verifier admission;
- deterministic VM execution;
- disassembly readability.

## Proposed layout

```text
examples/canonical/weather_station/
  weather_station.sm
  README.md

tests/golden/weather_station.sm
# generated or checked-in policy decision:
tests/golden/weather_station.smc
```

The `.smc` artifact policy must be explicit: either checked in as a golden binary or regenerated by snapshot tests.

## Required README sections

- Purpose;
- Quad status model: `T/F/S/N`;
- Sensor validation rules;
- Test cases table;
- Pipeline commands;
- Expected result;
- Known current language constraints used by the example;
- Future cleanup once line endings/comments/f64 relational operators are hardened.

## Required commands

```powershell
smc check examples/canonical/weather_station/weather_station.sm
smc lint examples/canonical/weather_station/weather_station.sm
smc compile examples/canonical/weather_station/weather_station.sm -o target/weather_station.smc --opt-level O1
smc verify target/weather_station.smc
smc run-smc target/weather_station.smc
smc disasm target/weather_station.smc
```

## Acceptance criteria

- example source is committed in a canonical examples path;
- source passes `smc check` and `smc lint`;
- compiled SemCode verifies;
- `run-smc` exits successfully;
- README documents the example and current constraints;
- golden/snapshot policy is explicit;
- example does not depend on UI/Workbench.

## Out of scope

- implementing missing frontend features;
- adding UI output;
- adding weather data IO;
- changing release status;
- replacing the current helper workarounds before the corresponding gaps are fixed.

## Follow-up cleanup after related gaps

After line-ending/comment/f64-relational fixes land, update the example to:

- allow ordinary comments;
- avoid CRLF-only assumptions;
- replace `gt` / `lt` helper workaround with direct operators if admitted.
